

## خطة تطوير نقطة بيع الوقود والمضخات والخزانات

### 1. إلزام اختيار لوحة في نقطة البيع
- إضافة حقل اختيار رقم اللوحة (من لوحات العميل المحدد) بعد اختيار العميل
- إضافة state جديد `selectedPlate`
- عند اختيار عميل نوعه شركة/حكومي، يظهر حقل اختيار اللوحة من اللوحات المسجلة
- يُمنع تأكيد البيع بدون اختيار لوحة (إذا كان العميل محدداً)

### 2. إصلاح رصيد المحفظة في POS
- **السبب**: البيانات من الـ API ترجع `fuel_wallets` ككائن وليس مصفوفة: `{"id": "...", "balance": 1000}`
- الكود الحالي يستخدم `fuel_wallets?.[0]?.balance` — خطأ
- **الإصلاح**: تغيير إلى `fuel_wallets?.balance` و `fuel_wallets?.id`

### 3. عرض رصيد المضخة (الخزان المربوط) في POS
- تعديل استعلام المضخات ليجلب بيانات الخزان: `fuel_pumps.select("id, pump_number, fuel_type, tank_id, fuel_tanks(tank_name, current_qty, capacity)")`
- عند اختيار مضخة، عرض بطاقة معلومات تحتوي على: اسم الخزان، الكمية الحالية، السعة، ونسبة الامتلاء بـ Progress bar

### 4. تحويل المضخات والخزانات لعرض بطاقات
- **FuelPumps.tsx**: تحويل من DataTable إلى شبكة بطاقات (grid cards) — كل بطاقة تعرض رقم المضخة، نوع الوقود، الخزان المربوط، قراءة العداد، والحالة
- **FuelTanks.tsx**: تحويل لبطاقات — كل بطاقة تعرض اسم الخزان، نوع الوقود، Progress bar للمستوى، حد التنبيه، والحالة مع زر التعبئة

### الملفات المعدّلة
- `src/pages/client/fuel/FuelPOS.tsx` — إصلاح المحفظة + لوحة + رصيد المضخة
- `src/pages/client/fuel/FuelPumps.tsx` — تحويل لبطاقات
- `src/pages/client/fuel/FuelTanks.tsx` — تحويل لبطاقات
