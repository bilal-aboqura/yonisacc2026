

## تطوير شاشة الخصومات: قيد يومية + تقسيم على أشهر + إجراءات كاملة

### 1. تعديل قاعدة البيانات (Migration)
إضافة أعمدة جديدة لجدول `hr_deductions`:
- `target_month` (integer) — شهر تطبيق الخصم في مسير الرواتب
- `target_year` (integer) — سنة تطبيق الخصم
- `installment_number` (integer, default 1) — رقم القسط
- `total_installments` (integer, default 1) — عدد الأقساط الإجمالي
- `journal_entry_id` (uuid, FK → journal_entries) — ربط مباشر بالقيد المحاسبي

### 2. قيد يومية تلقائي عند إنشاء خصم
نفس نمط السلف (Loans):
- التحقق من ربط الموظف بحساب في دليل الحسابات
- التحقق من وجود حساب مصروفات الخصومات في إعدادات HR
- إنشاء قيد: **مدين** حساب مصروفات الخصومات، **دائن** حساب الموظف
- حفظ `journal_entry_id` في سجل الخصم للربط المباشر

### 3. تحديد شهر الخصم وتقسيمه
- إضافة حقول "شهر التطبيق" و"سنة التطبيق" في نموذج الإضافة
- حقل "عدد الأقساط" (1-12): عند اختيار أكثر من 1، يتم تقسيم المبلغ وإنشاء سجلات متعددة
  - مثال: خصم 3000 على 3 أشهر → 3 سجلات × 1000 (كل واحد بشهر مختلف)
- مسير الرواتب يستخدم `target_month` و `target_year` لتحديد الخصومات المستحقة

### 4. إجراءات الجدول (عرض - تعديل - حذف)
- **قائمة DropdownMenu** لكل سطر (مثل شاشة السلف)
- **عرض**: Dialog يعرض كل التفاصيل (الموظف، النوع، المبلغ، الشهر المستهدف، رقم القسط، الحالة، القيد)
- **تعديل**: إعادة فتح النموذج بالبيانات الحالية + updateMutation
- **حذف**: 
  - `is_applied = true` → **منع الحذف** مع رسالة "لا يمكن حذف خصم تم تنفيذه في مسير رواتب"
  - `is_applied = false` → حذف القيد المحاسبي المرتبط (عبر `journal_entry_id`) ثم حذف الخصم

### 5. الملفات المتأثرة
- **Migration**: إضافة أعمدة `target_month`, `target_year`, `installment_number`, `total_installments`, `journal_entry_id`
- **`src/pages/client/hr/Deductions.tsx`**: إعادة بناء كامل مع الميزات الجديدة

### تدفق الحذف
```text
حذف خصم
  ├─ is_applied = true? → رفض "خصم مرتبط بمسير رواتب"
  └─ is_applied = false
       ├─ journal_entry_id موجود?
       │    ├─ حذف journal_entry_lines
       │    └─ حذف journal_entries
       └─ حذف hr_deductions
```
